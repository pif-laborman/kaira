## Codebase Patterns
- Data fixtures live in `src/data/` with barrel re-export from `src/data/index.ts`
- Each data file exports an explicit TypeScript interface and a named const array
- Tests use `node:test` + `node:assert/strict`, run with `npm test` (which runs `node --import tsx --test`)
- Test imports MUST use explicit file extensions (`.ts`, `.tsx`) for Node ESM resolution
- tsx is installed as a dev dependency for running TS tests
- Next.js 16 App Router with `src/app/` for routes
- Landing page served from `public/landing/` via middleware rewrite on `/`
- TypeScript strict mode, path alias `@/*` maps to `./src/*`
- Design system tokens defined in DESIGN.md, CSS variables in globals.css
- App routes use (app) route group with shared layout that includes BottomNav + AppProvider
- Components live in `src/components/`, context in `src/context/`
- App home is at `/home` (not `/`, which is the landing page via middleware rewrite)
- Fonts: Hanken Grotesk (--font-sans) and JetBrains Mono (--font-mono) loaded via next/font
- BottomNav is a fixed bottom bar, pages need paddingBottom: 80 to avoid overlap
- CarouselCounter component in `src/components/CarouselCounter.tsx` for mono fraction + nav buttons
- ScrollSection pattern: encapsulate useRef + useState in a component to avoid react-hooks/refs lint errors
- Horizontal scroll containers: use `overflowX: auto`, `scrollSnapType: x mandatory`, `scrollbarWidth: none`
- Pages using `useSearchParams` must wrap content in `<Suspense>` for Next.js 16 static generation
- Reusable SplitPill component in `src/components/SplitPill.tsx` (label + arrow, hover animation, sm/inverted variants)
- Reusable GhostButton component in `src/components/GhostButton.tsx` (1px border, uppercase, hover flood)
- Reusable Pill component in `src/components/Pill.tsx` (default/dark/outline variants, active state toggle)
- FilterPanel slide-over in `src/components/FilterPanel.tsx` (search, category pills, chakra pills, clear/done actions)
- Filter state pattern: local state in panel, applied on Done, parent filters pose list with intersection logic
